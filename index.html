Cheatsheet Project: Semantic Elements

Introduction

Semantic HTML is the application of HTML markup that reinforces the semantics, or meaning, of the information of the webpage rather than define its presentation or look. Semantic HTML is an important component of webpages; not only does it provide a browser context as to what is between it’s tags, but it also meets compliance standards for those with disabilities accessing pages through screen readers. Finally, applying semantic HTML can help improve a page’s search performance from a Search Engine Optimization perspective, making your webpage more visible to the world.

Primary Semantic Elements

HTML Tag Title Tag Description
<article> Article Defines an article, blog or news section
<aside> Aside Defines content related to <main>
<audio> Audio Defines audio content
<embed> Embed Defines externally linked content
<figcaption> Figure Caption Describes media in the <figure>
<figure> Figure Describes any media type
<footer> Footer Defines the bottom content of the page
<header> Header Defines the top content of the page
<main> Main Defines the primary content of the page
<nav> Navigation Defines navigation links
<section> Section Defines a specific page section
<video> Video Defines video content

Additional Semantic Elements

HTML Tag Title Tag Description
<data> Data Defines content to be read by virtual machine-reader
<details> Details Defines additional details to a webpage
<dialouge> Dialouge Defines a dialouge box or window
<summary> Summary Defines a visible heading for details
<time> Time Defines dates and times

Conclusion

There are many benefits to semantic HTML elements – including optimized readability for the user, understanding for developers, and optimization for search engines and webpage rankings. While there are more semantic HTML tags available to use in HTML documents, these tags are among the most common elements developers will come across. In the future, try applying semantic code to a webpage and explore its functionality.
